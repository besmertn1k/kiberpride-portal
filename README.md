# 🛡️ КиберPride — Портал сотрудников

Внутренний веб-портал для сотрудников компьютерного клуба **КиберPride**. Авторизация по email/паролю, управление объявлениями, журнал инцидентов, справочник правил — всё в одном месте.

## Возможности

- **Авторизация** — вход по email и паролю через Firebase Authentication
- **Роли** — старший админ (создаёт контент и аккаунты) и админ (читает, подтверждает)
- **Памятка** — карточки с основными правилами работы в клубе
- **Важное** — лента объявлений с подтверждением прочтения (видно кто ознакомился)
- **Журнал инцидентов** — фиксация нарушений с уровнем серьёзности (высокий/средний/низкий)
- **Чек-листы** — ссылки на онлайн чек-листы (клавиатуры и т.д.)
- **Управление сотрудниками** — старший создаёт аккаунты прямо через портал
- **Синхронизация** — данные обновляются каждые 15 секунд через Firebase
- **Адаптивность** — работает на ПК, планшете и телефоне

## Роли

| Роль | Возможности |

## Установка

### 1. Firebase проект

Используется существующий проект Firebase с Realtime Database и Authentication.

Необходимые настройки:
- **Authentication** → Способ входа → Email/Password → Включён
- **Realtime Database** → Правила:

```json
{
  "rules": {
    ".read": "auth != null",
    ".write": "auth != null"
  }
}
```

### 2. Первый пользователь

В Firebase Console → Authentication → Пользователи → **Добавить пользователя** (email старшего админа). При первом входе профиль создаётся автоматически с ролью `senior`.

### 3. Деплой

1. Переименуй `portal.html` в `index.html`
2. Создай репозиторий на GitHub
3. Загрузи файл
4. Settings → Pages → Branch: main → Save
5. Ссылка появится через минуту

### 4. Создание аккаунтов сотрудникам

Старший админ заходит в раздел **«Сотрудники»** в портале → заполняет имя, email, пароль → нажимает «Создать». Сотрудник получает логин и пароль для входа.

## Структура данных в Firebase

```
/portal
  /users
    /{uid} → { name, email, role }
  /announcements
    /{id}  → { id, text, author, ts, readers: { имя: true } }
  /incidents
    /{id}  → { id, sev, who, title, desc, author, ts }
```

## Безопасность

- Доступ к базе данных только для авторизованных пользователей
- Публикация контента ограничена ролью `senior`
- Пароли хранятся в Firebase Authentication (не в базе данных)
- API-ключ Firebase безопасен для клиентского использования (ограничен правилами)
